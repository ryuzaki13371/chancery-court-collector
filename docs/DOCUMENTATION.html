1 · Introduction & Background

Lead Collector is a set of three automations that gather sales-lead information from public Hamilton County, Tennessee government records, clean and organize it, and deliver it automatically to a Google Sheet and a Telegram bot.

The client, Steven R., works with government-records data and needed a simple, low-cost, repeatable way to collect it — originally framed around the no-code tool Browse AI, but with the explicit allowance to use custom code where it is “the best option.” In practice, the government sources involved (an ASP.NET search form and several PDFs) are beyond what a no-code scraper handles reliably, so the project was built as small, dependable scripts that run free in the cloud — meeting the client’s true priorities: reliable, lowest-cost, reusable.

All data sources are public government records. The automations read publicly available pages and documents — obituary listings, the County Trustee property search, Chancery Court dockets, and the Delinquent Tax Sale list.

2 · The Client’s Request

Three separate jobs were requested.

Task A — Obituaries → Property Addresses

  • Collect obituary names from chattanoogan.com/obituaries
  • Enter each name in “Last Name First Name” format (e.g. Robert Jones → Jones Robert; shortened forms like Jones Rob / Jones R. acceptable)
  • Search each name on the HC Trustee Property Tax site; if a result is found, collect the property address (e.g. 508 N HAVEN LN)
  • Add everything to a Google Sheet; run once per week

Task B — Chancery Court Docket Names

  • From hamiltontn.gov/ChanceryCourt_Dockets.aspx, open “Current Motion Call Docket Part 1” and “Part 2” PDFs
  • Extract the individual case-party names
  • Do not use the “Procedural Steps List” or “Motion Call Schedule” links — only the current motion call dockets contain case names

Task C — Delinquent Tax Sale → Owners

  • From the Delinquent Tax Sale List PDF, scrape the property addresses
  • Search each address on the Trustee Property Search site to identify the current owner

4 · The Three Automations

4.1 · Obituaries → Property Addresses Task A

Script: ObituaryPropertyCollector.py  ·  Sheet tab: Obituaries

  1. Reads the obituary list from chattanoogan.com/obituaries (≈20 most recent).
  2. Reformats each name to “Last First”: strips nicknames (“Don”), parentheticals, suffixes (Jr./Sr./III), and middle initials; flags nicknames & compound names for review.
  3. Searches the Trustee site by last name, then matches the right person by first name: full first name = “Matched”; surname + first initial only = “Possible — verify”; a very common surname+initial is skipped to avoid noise.
  4. Captures the property address (e.g. 302 BRADY POINT RD).

Columns: Collected On · Obit Date · Obituary Name · Search Name (Last First) · Property Address · Other Possible Addresses · Match · Notes · Source Link.

Expect blanks. The Trustee site lists only current owners; a deceased person often is no longer the owner of record (spouse / trust / sold / rented). Blank rows are the data reality, not a malfunction — this is exactly why the request said “IF the search returns a result.”

4.2 · Chancery Court Docket Names Task B

Script: ChanceryCourtCollector.py  ·  Sheet tab: Chancery Court

  1. Opens the Chancery Court Dockets page and finds the links by their label — “Current Motion Call Docket Part 1” & “Part 2” — so it always grabs the current week (no hard-coded, expiring links).
  2. Deliberately skips the “Procedural Steps List” and “Motion Call Schedule” links.
  3. Downloads both PDFs, reads them with pdftotext -layout, and extracts party names from the left (party) column, filtering out attorneys, firms, agencies, and boilerplate legal phrases.

Columns: Docket · Docket Date · Name.

4.3 · Delinquent Tax Sale → Owners Task C

Script: DelinquentTaxCollector.py  ·  Sheet tab: Tax Sale

  1. Downloads the current Delinquent Tax Sale List PDF (auto-detects the year).
  2. Parses each property: address, parcel, minimum bid, and whether it is already PAID (redeemed). Handles both city and rural parcel formats.
  3. Searches each address on the Trustee site and, using the exact parcel from the list, pins the correct owner — even on multi-unit addresses (e.g. a 508-unit condo).
  4. Sorts active leads first (still going to sale), redeemed below.

Columns: Sale Status · Property Address · Minimum Bid · Owner (from Trustee) · Lookup · Parcel · Docket · Item # · Collected On.

6 · Tutorial A — For the Client (Steven)

Getting your leads

  1. The Google Sheet fills itself — just open it. Three tabs: Chancery Court, Obituaries, Tax Sale.
  2. From your phone, any time: open t.me/LeadCollectorHQ_bot → press Start → tap a button. The file (PDF + spreadsheet) arrives in the chat in 1–2 minutes.
  3. Weekly: once you’ve pressed Start, a fresh copy is sent to you automatically every Monday. Send /stop to pause, /start to resume.

Reading the results

  • Match = Matched → one confident address. Possible — verify → likely, check it. No match / Common name — skipped → no reliable address (see §11).
  • Tax Sale → Sale Status: “Active — going to sale” are the live leads (sorted to the top); “Redeemed (paid)” have been settled.
Nothing to install, no password to manage, no cost to you. Open the sheet, or tap the bot. That’s it.

7 · Tutorial B — For the Operator

Run a collector manually (cloud)

GitHub → repo → Actions tab → pick a workflow → Run workflow. Or via CLI:

gh workflow run weekly.yml      --repo ryuzaki13371/chancery-court-collector   # dockets
gh workflow run obituaries.yml  --repo ryuzaki13371/chancery-court-collector
gh workflow run taxsale.yml     --repo ryuzaki13371/chancery-court-collector

Run a collector locally

pip install requests reportlab google-auth
sudo apt-get install -y poppler-utils
python3 ChanceryCourtCollector.py        # or ObituaryPropertyCollector.py / DelinquentTaxCollector.py
python3 tools/csv_to_pdf.py result.csv out.pdf "Title"

Point it at a different Google Sheet (e.g. the client’s own)

  1. Client creates a Google Sheet, clicks Share, adds the service-account email as Editor: lead-collector@lead-collector-500213.iam.gserviceaccount.com
  2. Client sends the sheet link.
  3. Update the GitHub secret: gh secret set SHEET_ID --repo … (paste the new id).

Add another client / recipient

They open t.me/LeadCollectorHQ_bot and press Start — that auto-subscribes their chat (recorded in subscribers.txt). Weekly runs then deliver to them too.

8 · Tutorial C — Setup & Deployment (from scratch)

For rebuilding the system in a fresh environment. Four parts.

8.1 · GitHub repository & Actions

  1. Create a repo and push the project files.
  2. Add repository Secrets (Settings → Secrets and variables → Actions): TELEGRAM_TOKEN, TELEGRAM_CHAT_ID, GCP_SA_KEY, SHEET_ID.
  3. The workflows in .github/workflows/ run automatically; public repos get unlimited free Actions minutes.

8.2 · Telegram bot (BotFather)

  1. In Telegram, message @BotFather → /newbot → name it → get the bot token.
  2. Set the command menu with setMyCommands; set the description with setMyDescription.

8.3 · Cloudflare Worker (bot brain)

  1. Create a free Cloudflare account.
  2. From telegram-bot/: npx wrangler deploy.
  3. Set Worker secrets: BOT_TOKEN, GH_TOKEN (fine-grained, Actions:read&write), WEBHOOK_SECRET; the GH_REPO var lives in wrangler.toml.
  4. Point Telegram at the Worker: setWebhook?url=<worker-url>&secret_token=<WEBHOOK_SECRET>.

8.4 · Google service account & Sheet

  1. Google Cloud Console → new project → enable the Google Sheets API.
  2. Create a service account, download its JSON key → store as the GCP_SA_KEY secret.
  3. Create a Google Sheet, Share it (Editor) with the service-account email, put its id in SHEET_ID.
Full bot deploy steps are also in telegram-bot/DEPLOY.md in the repo.

11 · Data Quality & Honest Notes

  • Obituary blanks are normal. Many deceased aren’t the current owner of record; those rows are correctly blank and labelled. A typical week yields addresses for roughly half to two-thirds of names.
  • “Possible — verify” means verify. These matched on surname + first initial, so they may be a same-initial different person. Real address, human check advised.
  • Tax-sale “Redeemed (paid)” rows are already settled — the live opportunities are the “Active — going to sale” rows, sorted to the top.
  • Court dockets: the main party per case is captured reliably; an occasional extra co-party crammed mid-line may be missed. A handful of edge names can need a quick glance.
  • Open question for the client: for the tax-sale lookup we collect the owner name. If the mailing address or amount owed is also wanted from the Trustee detail page, that is a small addition.

12 · Security & Credentials

  • Secrets are never in the repo. All tokens/keys live in encrypted GitHub Actions secrets and Cloudflare Worker secrets. The repo is public but contains code only — lead data goes to the Sheet/Telegram, never committed.
  • Least privilege. The bot’s GitHub token is fine-grained (Actions on one repo only). Data workflows run read-only.
  • Credentials in use: Telegram bot token · GitHub fine-grained token · Google service-account key · webhook secret.
Housekeeping recommended. Any credential that was shared during setup should be rotated once the system is stable, and temporary build tokens (used only to deploy) should be revoked. None of this affects the running system.

13 · Maintenance & Adding New Sources

The engine — name/address formatting, owner matching, de-duplication, scheduling, and Sheet + Telegram + PDF delivery — is shared across all tasks. Adding a new source only requires a new “reader” for that page or PDF; everything downstream is reused.

To add a new source

  1. Write a small collector that outputs a clean CSV (model it on an existing one).
  2. Add a workflow (copy an existing one; set the tab name + dedup key).
  3. Add a button + command in telegram-bot/worker.js and redeploy the Worker.

Routine upkeep

  • If a county page changes layout, the matching “reader” may need a small tweak.
  • If a run shows “Site error”, simply re-run — the Trustee site occasionally times out.

17 · Glossary

TermMeaning
Trustee siteHamilton County property-tax search (tpti.hamiltontn.gov) — maps a name/address to the current owner & property.
ParcelThe county’s unique id for a piece of land (e.g. 117O-A-011 C158) — used to pin the exact unit at a shared address.
Motion Call DocketThe Chancery Court list of cases being heard — the only docket files that contain party names.
Delinquent Tax SaleProperties with unpaid taxes scheduled for auction; “Redeemed/PAID” means the owner settled and it’s off the sale.
Service accountA “robot” Google account the automation uses to write into a shared Sheet.
GitHub ActionsGitHub’s free cloud runner — executes the collectors on a schedule and on demand.
Cloudflare WorkerA tiny always-on cloud program that powers the bot’s buttons; free.
De-duplicationSkipping rows already present (by a key column) so weekly runs grow the list without repeats.
